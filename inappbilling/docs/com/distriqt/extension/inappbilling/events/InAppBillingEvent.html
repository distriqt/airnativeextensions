CONSUME_FAILEDConstant
public static const CONSUME_FAILED:String = consume:failed

Dispatched when a consume purchase call fails, either due to communication problems or some other internal error.

The #message parameter will contain a message describing the error.

CONSUME_SUCCESSConstant 
public static const CONSUME_SUCCESS:String = consume:success

Dispatched when a consume purchase has been completed and is successful.

The #data contains an array with one element, being the details of the purchase, the only garanteed value here is the productId, the others may be invalid.

INVALID_PRODUCTConstant 
public static const INVALID_PRODUCT:String = product:invalid

This error occurs when there was an invalid product ID requested. If this error occurs you should check that you have correctly specified a product ID and that it matches an ID that was setup in the store.

When this occurs you will not be able to purchase the product specified by the ID.

In this case the #errorCode variable will contain the invalid ID.

PRODUCTS_FAILEDConstant 
public static const PRODUCTS_FAILED:String = products:failed

Dispatched when an attempt to retrieve the products fails, after a call to getProducts. This is generally due to a failure to connect to the specified AppStore due to connectivity problems.

When this happens you are advised to wait a small interval and retry the call. You may also wish to check if the user has a valid network connection using the NetworkInfo ANE or other such method and inform the user of the problem.

PRODUCTS_LOADEDConstant 
public static const PRODUCTS_LOADED:String = products:loaded

Dispatched when the specified products have been successfully loaded from the server after a call to getProducts.

The #data parameter will contain the array of products successfully loaded

PURCHASE_SUCCESSConstant 
public static const PURCHASE_SUCCESS:String = purchase:success

A purchase was successful. When you receive this event you should deliver the product to the application.

You will receive this as the result of a restore, and after a purchase was made, for each individual successful purchase.

The #data contains an array with one element, being the details of the purchase.

RESTORE_PURCHASES_FAILEDConstant 
public static const RESTORE_PURCHASES_FAILED:String = restore:purchases:failed

Indicates there was an error with the restore purchases process.

The #errorCode and #message will be set accordingly to show further information about the error.

RESTORE_PURCHASES_SUCCESSConstant 
public static const RESTORE_PURCHASES_SUCCESS:String = restore:purchases:success

The restore purchases process has been completed and was successful.

There is no data attached to this event, instead each of the individual purchase events will be fired again when the restore purchases process is called.
