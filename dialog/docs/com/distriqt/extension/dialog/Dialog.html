Classpublic class Dialog
InheritanceDialog com.distriqt.extension.dialog.base.ExtensionBaseImpl

This class represents the dialog extension.

Dialogs allow you to present a simple question or choice to the user in a modal manner. These alerts and other dialogs can be stacked and must all be dismissed by the user.

dismissDialog()method 
public function dismissDialog(id:int):Boolean

Dismiss a dialog that you have previously displayed.

The id of the dialog must be provided so that if you have stacked dialogs the correct dialog will be dismissed.

This does not dismiss progress dialogs only dialogs created with the #showAlertDialog or #showMultipleChoiceDialog functions.

If called at the appropriate time, you will receive a DialogEvent.DIALOG_CANCELLED event after this dialog is dismissed.

iOS: Under iOS the dialogs take a little time to initialise so make sure you leave at least 1-2 seconds between creating the dialog and dismissing it. If you don't you may not correctly get the any other dialogs reappearing and you won't receive the cancelled event. The dialog however will still be dismissed if you don't wait.

Parameters

id:int — The unique id of the dialog to dismiss (same as the id passed to the showialog functions)

Returns
Boolean — true if the dialog with the specified id was found and dismiss called and false otherwise.

See also

com.distriqt.extension.dialog.events.DialogEvent.DIALOG_CANCELLED

showDateTimePicker()method 
public function showDateTimePicker(id:int, options:DateTimePickerOptions = null, mode:String):Boolean

This is the main function for displaying a date and/or time picker dialog. There are three modes of operation as defined by the DateTimePickerOptions.MODE_constants.

These dialogs are presented modally with your application being dimmed in the background. You specify the options of the dialog by creating an instance of the DateTimePickerOptions class and setting the options there. See that class for more information on the available options.

Events are dispatched from the dialog, including change, selection and cancel events. The normal DialogEvent.DIALOG_CANCELLED event is fired when the dialog is cancelled by the user. The changed and selection events are in the DialogDateTimeEvent class.

  • MODE_TIME: Displays only the time selection in the picker dialog (hour, minute)
  • MODE_DATE: Displays only the date selection in the picker dialog (year, month, day)
  • MODE_DATE_AND_TIME: Displays both the date and time selection in the picker dialog simultaneously. iOS only
Android

Please note that on Android you can only display either the date or the time, so you'll have to accommodate this in your code. A simple option is to create both a date and time picker simultaneously and the user will have to interact with both before returning to your application.

Parameters

id:int — The id to assign to this dialog. It should be unique.
 
options:DateTimePickerOptions (default = null) — The display options for this date / time dialog
 
mode:String (default = NaN)

Returns
Boolean — true if the picker was successfully created and displayed and false if there was an error

See also

com.distriqt.extension.dialog.objects.DateTimePickerOptions

Example
The following example shows the simple usage to request a date input from the user. 
         Dialog.init( DEV_KEY );
         Dialog.service.addEventListener( DialogEvent.DIALOG_CANCELLED,      dialog_dialogCancelledHandler, false, 0, true );
         Dialog.service.addEventListener( DialogDateTimeEvent.DATE_CHANGED,  dialog_dateChangedHandler, false, 0, true );
         Dialog.service.addEventListener( DialogDateTimeEvent.DATE_SELECTED, dialog_dateSelectedHandler, false, 0, true );
         
         var options:DateTimePickerOptions = new DateTimePickerOptions();
         options.year = 2014;
         options.month = 10;
         options.day = 5;
         options.title = "Select a date";
         
         Dialog.service.showDateTimePicker( 1, options, DateTimePickerOptions.MODE_DATE );
         
         ...
         
         private function dialog_dialogCancelledHandler( event:DialogEvent ):void
         {
             trace( "The dialog was cancelled by the user" );
         }
         
         private function dialog_dateChangedHandler( event:DialogDateTimeEvent ):void
         {
             trace( "The date was changed" );
         }
         
         private function dialog_dateSelectedHandler( event:DialogDateTimeEvent ):void
         {
             trace( "Dialog date selected: " + event.date.toDateString() );
         }

Displays a multiple choice Dialog.

showProgressDialog()method 
public function showProgressDialog(id:int, title:String, message:String, style:String, cancelable:Boolean = true, theme:String):Boolean

Creates a progress dialog, to show progress of a process. Only one progress dialog can be shown at a time. If you attempt to create two, this function will return false. In this case you must call dismissProgressDialog before attempting to create another.

There are two styles of progress dialogs, a spinner, for undetermined length processes and a horizontal progress bar.

If the progress dialog is cancelable, you will receive a DIALOG_CANCELLED event if the user cancels the progress dialog, generally by pressing the back button. Under iOS progress dialogs are not cancelable.

Parameters

id:int — An id to associate with this dialog, will be returned with closed/cancelled events
 
title:String — A title for the dialog. If an empty string is provided the dialog will have no title.
 
message:String — A message for the dialog. If an empty string is provided the dialog will have no message area.
 
style:String (default = NaN) — The style of the dialog (either DIALOG_PROGRESS_STYLE_SPINNER or DIALOG_PROGRESS_STYLE_HORIZONTAL) (default: DIALOG_PROGRESS_STYLE_SPINNER)
 
cancelable:Boolean (default = true) — If true, the dialog can be cancelled by the user. Android only. (default: true)
 
theme:String — A value from the DialogTheme class. Android only. (default: THEME_TRADITIONAL)

Returns
Boolean — true if progress dialog creation was successful and false otherwise. iOS will always return false.

See also

DIALOG_PROGRESS_STYLE_SPINNER
DIALOG_PROGRESS_STYLE_DETERMINATE
DIALOG_PROGRESS_STYLE_HORIZONTAL

showSelectPopover()method 
public function showSelectPopover(id:int, title:String, values:Array, options:PopoverOptions = null, selection:Array = null):void

Displays a popover with a select list. The popover is positioned and sized based on the options specified in the PopoverOptions parameter. The values contained in the select list should be a series of strings that the user can select.

Parameters

id:int — An id to assign to this popover
 
title:String — The title to display. (Note: you'll need to set the showNavigationBar in the options for this to be visible).
 
values:Array — An Array of Strings, each being an option in the select list.
 
options:PopoverOptions (default = null) — The options for position, size and display of the popover
 
selection:Array (default = null) — An array on integers that represent indexes into the values array of the selected items. This is only useful when you have a multi-select list and want to preselect some options. Generally it can be left as null or an empty array.

See also

com.distriqt.extension.dialog.objects.PopoverOptions

Example

The following example shows how to display a select list popover. 

         Dialog.init( DEV_KEY );
         Dialog.service.addEventListener( PopoverEvent.POPOVER_CLOSED, popover_closedHandler, false, 0, true );
         Dialog.service.addEventListener( PopoverEvent.POPOVER_CHANGE, popover_changeHandler, false, 0, true );
         
         var options:PopoverOptions = new PopoverOptions();
         options.position = new Rectangle( 50, 250, 150, 50 );
         options.size = new Rectangle( 0, 0, 300, 350 );
         options.arrowDirection = PopoverOptions.ARROW_DIRECTION_ANY;
         
         Dialog.service.showSelectPopover( 1, "Genres", ["All", "Punk", "Rock", "ATTACK OF THE CLONES"], options, [ 1 ] );
         
         ...
                  private function popover_changeHandler( event:PopoverEvent ):void
         {
             trace( "The user changed the selection in the popover" );
         }
         
         private function popover_closedHandler( event:PopoverEvent ):void
         {
             // The selection will be the array of indexes into your values that were selected by the user
             trace( "Popover Closed: "+event.id +" selection: "+event.selection.join(",") );
         }
