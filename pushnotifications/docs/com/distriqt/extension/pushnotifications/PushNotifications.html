Classpublic class PushNotifications
InheritancePushNotifications com.distriqt.extension.pushnotifications.base.ExtensionBaseImpl

This class is the Push Notifications Native Extension interface. It implements the different push notifications services on the supported platforms giving the developer one consistent interface to push notifications across these platforms.

The currently implemented platforms are:

  • Apple Push Notification Service (APNS) on iOS
  • Google Cloud Messaging (GCM) on Android

We have implemented a similar approach on both iOS and Android and it follows the following methodology. There are three main states of running an application: foreground execution when the application is running in the foreground, background execution when the application is running but not in the foreground, and when the application is not running at all. In each state the following occurs:

  1. Foreground: No notification is displayed in the notification area, you'll immediately receive the notification event. If you wish to display a notification in this circumstance you can use the Notifications extension to do so.
  2. Not running: A notification is displayed in the notification area. When clicked the application is started. When you call register() you'll immediately receive the notification event.
  3. Background: A notification is displayed in the notification area. When the user clicks on the notification the application will be brought into focus and you'll receive the notification event.
Apple Push Notifications

Please see our tutorial on setting up the APNS here: http://distriqt.uservoice.com/knowledgebase/articles/252914-push-notifications-tutorial-part-1

Google Cloud Messaging

There are several steps in order to get an application running using GCM. The "Getting Started" documentation here is the best place to start: http://developer.android.com/guide/google/gcm/gs.html

Briefly the steps are as follows:

  • Open the Google APIs Console page.
  • Create an API Project and take note of the project number, this will become your "GCM Sender ID" that you must pass to the register function
  • Enable the GCM Service in your projects Services section on the Google API's Console page

These are the minimum steps required, the rest pertain to getting your server setup to send messages to your clients. There are some tutorials on our site on how to get different technologies to send messages with GCM.

Automatic Notifications

Under Android there is no "default" way of displaying the notifications. We have implemented a simple message that will get displayed in the notification area when you supply certain variables in the message content.

In order to display a notification automatically when your notification arrives you must specify the some specific variables in the GCM data packet. These are as follows:

  • tickerText: The text displayed in the ticker bar along the top of the device
  • contentTitle: The title of the notification in the notification centre
  • contentText: The text displayed in the notification centre below the title
  • sound: If set to default plays the default sound on notification arrival
  • vibrate: If set to true vibrtes the device on notification arrival
  • id: Optional. If set then you can specify the id of the notification. Sending the same notification id will update an existing notification rather than producing another one.

For example: 

	 "data" : { 
	     "tickerText" : "some ticker text", 
	     "contentTitle" : "some content title",
	     "contentText" : "some content description text",
	     "sound" : "default",
	     "vibrate" : "false",
	     "otherCustomData" : "some value" 
	 },
	 "registration_ids" : { ... }

Note: to have the vibration work you need to add the vibration permission to your application manifest additions:

	 <uses-permission android:name="android.permission.VIBRATE"/>
Notes and Limitations

Update: As of version 2 the push notifications will be dispatched once when you call register. It's suggested you make sure you've added your notification listeners before calling register so you correctly receive this event.

This is for both iOS and Android! That is there is no longer a limitation on the GCM service.

An important note here is that the GCM Service doesn't receive notifications if the application is "forced stopped". This means that when the application is either:

  • first installed, before the user manually invokes the application
  • after a user force stops the application until the user manually invokes the application agaion

This is by design of the notifications on the Android platform and not a limitation or restriction of this ANE. So if a user force stops your application, it will stop receiving push notifications. This is different from normal application exits and your application will receive notifications if not running after a normal application exit.

Android

The application descriptor additions under Android are located in a node called “android”. It should look something like the following. Please note the last receiver is a recent addition and you'll need to make sure you update appropriately. 

	 <android>
	 	<manifestAdditions><![CDATA[
	 		<manifest android:installLocation="auto">
	 	
	 			<uses-sdk android:minSdkVersion="8" android:targetSdkVersion="16"/>
	 
	 			<uses-permission android:name="android.permission.INTERNET"/>
	 			<uses-permission android:name="android.permission.GET_ACCOUNTS" />
	 			<uses-permission android:name="android.permission.VIBRATE"/>
	 			<uses-permission android:name="android.permission.WAKE_LOCK" />
	 			<uses-permission android:name="com.google.android.c2dm.permission.RECEIVE" />
	 
	 			<!-- Only this application can receive the messages and registration result --> 
	 			<permission android:name="air.com.distriqt.test.permission.C2D_MESSAGE" android:protectionLevel="signature" />
	 			<uses-permission android:name="air.com.distriqt.test.permission.C2D_MESSAGE" />
	 
	 			<application>
	 				<receiver android:enabled="true" android:exported="true" android:name="com.distriqt.extension.pushnotifications.PushNotificationsBroadcastReceiver" android:permission="com.google.android.c2dm.permission.SEND" >
	 					<intent-filter>
	 						<action android:name="com.google.android.c2dm.intent.RECEIVE" />
	 						<action android:name="com.google.android.c2dm.intent.REGISTRATION" />
	 						<category android:name="air.com.distriqt.test" />
	 					</intent-filter>
	 				</receiver>
	 				<service android:enabled="true" android:exported="true" android:name="com.distriqt.extension.pushnotifications.gcm.GCMIntentService" />
	 				<receiver android:name="com.distriqt.extension.pushnotifications.PushNotificationsReceiver">
	 					<intent-filter>
	 						<action android:name="air.com.distriqt.test.PUSHNOTIFICATION_SELECTED" />
	 					</intent-filter>
	 				</receiver>
	 			</application>
	 
	 		</manifest>
	 
	 	]]></manifestAdditions>
	 </android>

There are two important things here. If you are adding this to an existing application, you must make sure you have only one application node inside your manifest additions. You’ll need to copy the receivers and service from that node and place that inside your current application node. If you have two application nodes the push notifications may not work.

Secondly you need to replace all the application ID’s (air.com.distriqt.test) with your application ID. It is important here that you use the full application ID with the air. prefix as this will be your actual ID in the Android system.
