alertActionproperty
alertAction:String

The title of the action button or slider.

Assign a string as the value. The alert action is the title of the right button of the alert or the value of the unlock slider, where the value replaces “unlock” in "slide to unlock".

If you specify nil, and alertBody is non-nil, "View" (localized to the preferred language) is used as the default value.

Supported: iOS

playSoundproperty 
playSound:Boolean

Use the default notification sound.

If the notification will cause the default sound to play

Note: Under iOS this has some restrictions. The user can select the notification setting in their system settings so setting this to true will either:

  • make the phone vibrate if the silent mode is selected and vibrate enabled
  • make the phone play the default sound but only if the application is in the background
  • nothing if the phone is not in vibrate and not in the background

If the application is running and in the foreground, then you should play your own sound as nothing will be done by iOS.

Supported: Android, iOS

The default value is false.

soundNameproperty 
soundName:String

The file name of the sound file to play when the notification is fired. It must be a wav, caf, mp3 or other suitable encoded file, and must be less than 30 seconds in length.

The sound must be packaged as part of your application and reference the path relative to the main application file.

Flash builder normally will do this automatically when you add a sound, however you should double check under: Actionscript Build Packaging / [Platform] / Package contents

iOS: Sounds that last longer than 30 seconds are not supported. If you specify a file with a sound that plays over 30 seconds, the default sound is played instead.

playSound must be set to true

Supported: iOS, Android

The default value is default notification sound, empty string: "".
