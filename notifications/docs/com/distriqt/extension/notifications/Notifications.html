Packagecom.distriqt.extension.notifications
Classpublic class Notifications
InheritanceNotifications com.distriqt.extension.notifications.base.ExtensionBase

This extension allows access to native local notifications, as opposed to push notifications. These notifications are displayed locally in the notification area of the device, generally the pull down area at the top of the device.

Individual settings available for each platforms notifications are described in the documentation for the Notification class.

Android

Notifications dispatched will appear in the Notification Center

Under Android you can schedule notifications to be delayed, however be aware that this process can be interupted by a task killer which will terminate the application and remove any system alarms, or delayed notifications associated with the applicaiton.

If you want to receive delayed notifications when your application has stopped running or is running in the background you must add the following line into the manifest additions of your application descriptor. This will allow the extension to restart your application and pass the information about the notification through to your handler. 

	 <receiver android:name="com.distriqt.extension.notifications.NotificationsReceiver">
	 	<intent-filter>
	 		<action android:name="air.YOUR_APP_ID.NOTIFICATION" />
	 		<action android:name="air.YOUR_APP_ID.DELAYED_NOTIFICATION" />
	 	</intent-filter>
	 </receiver>

Note: Please note the name change from the older version of the extension. You now need to use your application ID (prefix with air as shown). eg air.com.distriqt.text.NOTIFICATION

If your application has stopped running between the scheduling and dispatch of a delayed notification, you'll receive the "selected" event as soon as you call register so make sure you've added your handlers before calling register.

Note: There is currently no way to use custom icons in the extension. These appear in the notification center and the task bar under Android. These have to be compiled as part of the extension so we've only provided a small set of icons for use. You can however use your application icon (which is the default)

iOS

To make sure you can successfully set the badge icon number in your application make sure to check the "Badge App Icon" setting for your app is turned on in the Notification Center. Go to Settings > Notifications. Find your app either in "In Notification Center" section or "Not In Notification Center" section, and tap on it. Then turn on the "Badge App Icon" setting. p/s: it doesn't matter if you app is not in notification center (i.e. Notification Center = OFF); the badge icon setting does take effect nevertheless.

register()method 
public function register():void

Registers to receive notifications.

This is important mainly if you expect to receive startup notifications. If the application was started via a local notification, you will receive a notification selected event immediately after calling this function so make sure you've added a listener for this event prior to calling this function.
