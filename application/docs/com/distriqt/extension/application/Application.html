Packagecom.distriqt.extension.application
Classpublic class Application
InheritanceApplication com.distriqt.extension.application.base.ExtensionBase

This class represents the application extension.

It provides application options and settings that aren't available elsewhere through the AIR SDK.

Unique ID

Provides access to the unique device ID's available on the various platforms. 

     var deviceID:String = Application.service.device.uniqueId( Device.VENDOR );

Auto Start on Boot

This setting allows you to make this application automatically start when the device boots up. It is currently available on Android only.

To use this you have to add some definitions to your application descriptor manifest additions and then enable the auto start in the extension. The additions to your application descriptor are shown in the description of the setAutoStart() function below. Once you've added these additions the code you need to add to the actionscript side of your application is quite simple: 

     Application.init( DEV_KEY );
     Application.service.setAutoStart( true );

You can also use the isAutoStartEnabled() function to determine if autostart is already enabled in your application. 

     Application.init( DEV_KEY );
     if (!Application.service.isAutoStartEnabled())
         Application.service.setAutoStart( true );

Display Modes

These display modes allow you to control additional parts of the display UI. The main component here is the Android navigation bar, which on some versions of the OS shows the on-screen navigation controls: back, home, menu, search etc.

In addition we allow setting the traditional fullscreen and normal display modes through the native implementation.

For more information on the available modes and options see the setDisplayMode() function documentation and the ApplicationDisplayModes.

With the navigation UI component you can listen for ApplicationEvent.UI_NAVIGATION_CHANGED events dispatched to notify you when the state of the navigation UI component changes. This may be the result of a call to setDisplayMode() or as the result of a system interaction that forced the navigation UI component to change stage.

For example: 

     Application.service.addEventListener( ApplicationEvent.UI_NAVIGATION_CHANGED, application_uiNavigationChangedHandler, false, 0, true );
     Application.service.setDisplayMode( ApplicationDisplayModes.UI_NAVIGATION_HIDE );
     
     private function application_uiNavigationChangedHandler( event:ApplicationEvent ):void
     {
         trace( event.type + "::" + event.option );
     }
iOS

Requires at least iOS 6+ as the unique ID functionality uses the new SDK's provided in that release. Prior to this version it was required to do some hacks to generate a decent unique ID. If you require older versions of iOS please contact us for a custom ANE specific to your ID needs.

setAutoStart()method 
public function setAutoStart(enabled:Boolean):Boolean

Sets whether the applicaiton should start automatically when the device boots.

This stores a setting in the applications shared preferences to specify the application to start on boot.

Support: Android only

In order for this to work correctly you must add the following to your manifestAdditions node in your application descriptor file: 

		 <uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED" />
		 <application>
		     <receiver android:enabled="true" android:name="com.distriqt.extension.application.receivers.ApplicationStartupReceiver" android:permission="android.permission.RECEIVE_BOOT_COMPLETED">
		         <intent-filter>
		             <action android:name="android.intent.action.BOOT_COMPLETED" />
		             <category android:name="android.intent.category.DEFAULT" />
		         </intent-filter>
		     </receiver>
		 </application>

Parameters

enabled:Boolean — If true the application will automatically start on following device bootups, otherwise, if false the application won't autostart at boot.

Returns
Boolean — True if the setting was changed successfully or false if there was
setDisplayMode()method 
public function setDisplayMode(mode:String):void

Sets the display mode for the application

Note: With all the System UI Navigation changes there is a limitation: because navigation controls are so important, the least user interaction will cause them to reappear immediately. This means that you may use the hidden state of the navigation bar only when there is no user interaction, e.g. when watching a movie or other such activity.

Support: Android only
