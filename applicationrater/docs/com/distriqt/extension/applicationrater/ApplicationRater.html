Packagecom.distriqt.extension.applicationrater
Classpublic class ApplicationRater
InheritanceApplicationRater com.distriqt.extension.applicationrater.base.ExtensionBaseImpl

This class represents the Application Rater extension.

With this extension you can ask users of your application to return to the application stores and rate your application.

You must be careful to set the limits in this extension to be appropriate to your users, not to annoy them with repeated requests for ratings as this will often end up in a poor rating. The default operation waits 5 days to prompt the user.

There are several different conditions applicable to the prompt display:

  • days from initial launch
  • number of launches
  • number of significant user events

If you have set multiple conditions then the first condition to be met will cause the rate dialog to be displayed

Note: This extension stores the current launch count, events and settings in a file in the application directory.

Usage

The most basic usage will use the default settings to count the number of days since first launch and display the rate dialog after the max days is reached (default of 5 days).

The code to achieve this is quite simple, just add the following lines at some point during your application launch. We suggest somewhere after your is initialised and displayed. The important bit is to determine your application ID and correctly set that in the setApplicationId call (see the information in the documentation on the setApplicationId function). 

     ApplicationRater.init( DEV_KEY );
     ApplicationRater.service.setApplicationId( "air.com.distriqt.test", ApplicationRater.IMPLEMENTATION_ANDROID );
     ApplicationRater.service.setApplicationId( "552872162", ApplicationRater.IMPLEMENTATION_IOS );
     ApplicationRater.service.applicationLaunched();

If you set this to false, you'll need to manually call hasMetConditions to check if the conditions have been met and then showRateDialog when you wish to display the rate dialog

setApplicationId()method 
public function setApplicationId(applicationId:String, platform:String, applicationStore:String):void

Sets the application id associated with this rate dialog. This determines which application the user will be taken to in the application store.

This is the most important thing to set in the extension, as an invalid value will take you to the wrong application, or fail altogether. This ID is unfortunately different on the different operating systems. However you can call this function multiple times with the platform string specifying the platform for the applicationId so you won't have to determine the OS to set the correct ID. If you don't specify the platform then it is assumed you have determined to correct ID and it will overwrite any previous value.

Android: Under Android the value is the application identifier defined in your application descriptor, preceeded by air.. So, for example the distriqt test application com.distriqt.test has a Google Play id of air.com.distriqt.test. A simple way to attain this value is to use the NativeApplication class as follows:

var applicationId:String = "air." + NativeApplication.nativeApplication.applicationId;

iOS: Under iOS the value is a little more complicated. It's the value of the Apple ID in your iTunes Connect application page. For example, the distriqt test application com.distriqt.test has an Apple ID of 552872162. It can also be found in the iTunes App Store link URL which will be of the form https://itunes.apple.com/us/app/[APP_NAME]/id[APP_ID]

Default: The default value uses the value from the NativeApplication applicationId variable, preceeded by air. which should be the valid id for your Android application in the Google Play store.

Parameters

applicationId:String — The application ID string
 
platform:String — The platform implementation to set the ID for. Should use either:
  • ApplicationRater.IMPLEMENTATION_ANDROID
  • ApplicationRater.IMPLEMENTATION_IOS
  • ApplicationRater.IMPLEMENTATION_DEFAULT
Default: overwrites ID on all platforms
 
applicationStore:String (default = NaN) — The store that the extension should redirect to. If not set the default store will be used. This only has affect on Android as there is only one store on iOS. On Android this should be one of the STORE_types.


Example
In the following example we specify the application IDs on the two platforms in an application. On Android the first line will set the application ID but second line will do nothing internally. The reverse will happen on iOS, the first line will do nothing and the second will set the ID. 
         ApplicationRater.service.setApplicationId( "air.com.distriqt.test", ApplicationRater.IMPLEMENTATION_ANDROID );
         ApplicationRater.service.setApplicationId( "552872162", ApplicationRater.IMPLEMENTATION_IOS );
