Classpublic class AndroidRoot
InheritanceAndroidRoot com.distriqt.extension.androidroot.base.ExtensionBaseImpl

This class represents the Android Root extension. It provides access to some root only features on Android. These features are generally not supported by Google and will void warranty if enabled. Use this extension only if you are sure of what you are doing and understand advanced Android processes.

Your device must be unlocked and have root access, with busybox and superuser installed to use this extension. The functions in this extension should work on all Honeycomb, Ice Cream Sandwich and Jelly Bean devices.

Busybox must be installed and at least once run, a reboot is needed after Busybox has been run. It's also suggested that you grant the application su privilages and turn off notifications in SuperSU so that notifications aren't shown to the user.

How to root your device

This is an advanced procedure and is not covered in detail here. Some starting points are:

  • Nexus Root Toolkit: http://www.wugfresh.com/nrt/
  • http://forum.xda-developers.com/showthread.php?t=1766475
  • How to Root the Google Nexus 7: http://www.youtube.com/watch?v=wI0_ylxyrWA
  • How to Unroot the Google Nexus 7: http://www.youtube.com/watch?v=5HW_e99I5iM

setAutoStart()method 
public function setAutoStart(enabled:Boolean):Boolean

Sets whether the applicaiton should start automatically when the device boots.

This stores a setting in the applications shared preferences to specify the application to start on boot.

Support: Android only

In order for this to work correctly you must add the following to your manifestAdditions node in your application descriptor file: 

		 <uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED" />
		 <application>
		     <receiver android:enabled="true" android:name="com.distriqt.extension.androidroot.receivers.AndroidRootStartupReceiver" android:permission="android.permission.RECEIVE_BOOT_COMPLETED">
		         <intent-filter>
		             <action android:name="android.intent.action.BOOT_COMPLETED" />
		             <category android:name="android.intent.category.DEFAULT" />
		         </intent-filter>
		     </receiver>
		 </application>

Parameters

enabled:Boolean — If true the application will automatically start on following device bootups, otherwise, if false the application won't autostart at boot.

Returns
Boolean — True if the setting was changed successfully or false if there was
showSystembar()method 
public function showSystembar(makeVisible:Boolean, sleepTime:Number = 0.5):void

Shows or hides the System UI Navigation bar by killing any associated system ui processes.

Parameters

makeVisible:Boolean — If true the system UI is shown, otherwise it is removed.
 
sleepTime:Number (default = 0.5) — Sets number of seconds between calls to kill the navigation UI. This is required as the system may automatically restart the navigation UI. You'll need to experiment with this value based on your device. Modern devices will require a value ~0.5 but older devices can get away with 1-2 seconds.
