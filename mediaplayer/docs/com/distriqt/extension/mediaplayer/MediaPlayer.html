InheritanceMediaPlayer com.distriqt.extension.mediaplayer.base.ExtensionBaseImpl

This class represents the mediaplayer extension. This extension requires AIR 4.0 +

Initialising the extension

The extension needs to initialised once in your application. You should place this call somewhere during application start up or near where you plan to use the extension. Calling this function multiple times will result in an error.

This init call will throw an Error if any problems were encountered during the initialisation, so you should wrap this call in a try/catch and process the result appropriately for your application. 

     try
     {
         MediaPlayer.init( "" );
     }
     catch (e:Error)
     {
         // process the error
     }
Creating a player

The extension represents a single media player and allows you to control the media playback along with other things such as it's position and size.

You must start the media player by creating a player using the createPlayer function and specify the location of a media file to initially load into the player. You can also setup the player here, setting the size and location, whether the player auto starts media and the controls to display. 

     MediaPlayer.service.createPlayer( path, 0, 0, 640, 480, false, MediaPlayer.CONTROLS_EMBEDDED );
Controlling playback

Controlling the playback of the media is provided through the usage of the following functions:

  • MediaPlayer.service.play()
  • MediaPlayer.service.pause()
  • MediaPlayer.service.seek( time )
  • MediaPlayer.service.stop()

See the individual functions for more information on their operation.

Events

There are 3 main types of events associated with the media player: status, progress and errors.

Status events are dispatched using the MediaPlayerEvent class. These events include playback status, (playing, paused, stopped etc), player ready event, completion event, and fullscreen events.

You add a listener to the MediaPlayer.service to receive these events, for example to register for the READY events: 

     MediaPlayer.service.addEventListener( MediaPlayerEvent.READY, mediaPlayer_readyHandler );

Progress events are dispatched using the MediaProgressEvent class. This class includes the current playback position and the total time of the media. It's dispatched approximately 5 times a second.

Error events are dispatched using the MediaErrorEvent class. This event will contain more information on the error, but generally it's either due to incorrect file path or a network problem (for network streamed media).

Examples

This is the most simple example usage of the MediaPlayer extension. It firstly initialises the extension, then creates a 640x480 player and then calls play. In this example the mp4 file is included in the application package at the root level.

This example also shows the important function of the isSupported flag. 

     try
     {
         var path:String = File.applicationDirectory.nativePath + File.separator + "example.mp4";
     
         MediaPlayer.init( "" );
      if (MediaPlayer.isSupported)
         {
             MediaPlayer.service.createPlayer( path, 0, 0, 640, 480, false, MediaPlayer.CONTROLS_EMBEDDED ); 
             MediaPlayer.service.play();
         }
     }
     catch (e:Error)
     {
     }
The following example shows how to listen for ready, progress, error and complete events. 
     try
     {
         MediaPlayer.init( "" );
         MediaPlayer.service.addEventListener( MediaPlayerEvent.READY,         mediaPlayer_readyHandler, false, 0, true );
         MediaPlayer.service.addEventListener( MediaPlayerEvent.COMPLETE,     mediaPlayer_completeHandler, false, 0, true );
         MediaPlayer.service.addEventListener( MediaProgressEvent.PROGRESS,    mediaPlayer_progressHandler, false, 0, true );
         MediaPlayer.service.addEventListener( MediaErrorEvent.ERROR,         mediaPlayer_errorHandler, false, 0, true );
     
         var path:String = File.applicationDirectory.nativePath + File.separator + "example.mp4";
         MediaPlayer.service.createPlayer( path, 0, 0, 640, 480, false, MediaPlayer.CONTROLS_EMBEDDED ); 
     }
     catch (e:Error)
     {
     }

     private function mediaPlayer_readyHandler( event:MediaPlayerEvent ):void

     }

     }

     }

Resizing the player.

The player can be resized during operation. The following example resizes the player based on the Stage resize events. 

     try
     {
         MediaPlayer.init( "" );
     
         var path:String = File.applicationDirectory.nativePath + File.separator + "example.mp4";
         MediaPlayer.service.createPlayer( path, 0, 0, 640, 480, true, MediaPlayer.CONTROLS_EMBEDDED ); 
     
         stage.addEventListener( Event.RESIZE, stage_resizeHandler, false, 0, true );
     }
     catch (e:Error)
     {
     }
     
     
     ...
     
     
     private function stage_resizeHandler( event:Event ):void
     {
         MediaPlayer.service.resize( 0, 0, stage.stageWidth, stage.stageHeight );
     }
